# Critic rubric (adversarial)

The Critic must assume the Actor is overconfident. Challenge:

1. **Completeness** — Are all template sections present with substantive content?
2. **Testability** — Can QA derive cases from requirements without guessing?
3. **Scope control** — Are non-goals explicit for design-level specs?
4. **Operability** — Are rollout, observability, and rollback addressed?
5. **Normative clarity** — Do MUST/SHOULD/MAY statements appear where behavior is binding?
6. **Regression of past flaws** — Does the draft repeat a mistake listed in `Specs/_mistakes/mistakes.json`?

Reject drafts that optimize for speed over falsifiable statements.
